# Competition Manager 用户使用文档

本文档从普通管理员的视角说明如何启动和使用 Competition Manager。软件默认运行在你自己的电脑上，数据保存在本机 SQLite 文件中，不需要额外服务器。

## 1. 启动软件

### 1.1 第一次启动

在项目目录执行：

```bash
npm install
npm start
```

启动成功后，浏览器访问：

```text
http://127.0.0.1:3001
```

默认数据库文件位于：

```text
data/competition-manager.sqlite
```

备份文件位于：

```text
backups/
```

### 1.2 局域网访问

如果需要让同一 Wi-Fi 或局域网内的其他设备访问，在 `.env` 中设置：

```env
HOST=0.0.0.0
PORT=3001
```

然后其他设备访问：

```text
http://你的电脑局域网IP:3001
```

注意事项：

- 保持电脑不要睡眠。
- macOS 防火墙需要允许 Node.js 或终端网络访问。
- 管理操作需要登录管理员账号。

## 2. 首次初始化与登录

第一次打开页面时，左侧会显示管理员初始化区域。

1. 用户名默认是 `admin`，也可以修改。
2. 输入密码。
3. 点击 `初始化`。

初始化完成后：

- 左侧会显示当前登录用户。
- 新建项目、添加选手、创建赛事、备份等按钮会解除禁用。
- 以后再次打开页面时，使用同一账号密码登录即可。

如需退出，点击左侧 `退出`。

## 3. 项目管理

项目是最高隔离单位。不同项目之间的选手身份、赛事、Elo 分、WCC 分互不干涉。

### 3.1 创建项目

在左侧 `创建项目` 区域填写：

- `项目名称`：例如“国际象棋”“羽毛球单打”。
- `英文标识`：例如 `chess`、`badminton-single`，建议只使用英文、数字和连字符。

点击 `创建`。

创建成功后，顶部概览会显示：

- 项目名称
- 选手数量
- 赛事数量
- 当前最高 Elo

### 3.2 切换项目

在左侧 `当前项目` 下拉框选择项目。切换后页面会加载该项目的：

- 选手
- 赛事
- WCC 规则
- Elo 排名
- WCC 排名
- 综合排名

### 3.3 修改项目设置

选择项目后，左侧会显示 `项目设置`。

可修改：

- 项目名称
- 英文标识
- 默认 Elo
- Elo K 值
- 是否启用 Elo
- 是否启用 WCC

修改后点击 `保存项目`。

`重算积分` 用于在修正历史成绩、调整项目 Elo K 值或处理积分异常后，按当前比赛记录重新计算该项目内所有选手积分。

## 4. 选手管理

### 4.1 添加选手

在 `选手` 面板填写：

- `姓名`：真实姓名或主名称，必填。
- `显示名`：项目内显示名称，不填时通常可与姓名相同。
- `编号`：项目内编号，例如 `P001`。
- `昵称`
- `性别`
- `出生日期`
- `国家/地区`
- `地区/城市`
- `队伍/俱乐部`
- `联系方式`
- `头像链接`
- `种子序号`
- `备注`

点击 `添加`。

添加后，选手会出现在选手列表中，并显示：

- 当前 Elo
- 当前 WCC
- 种子序号
- 资料摘要

### 4.2 编辑选手

在选手列表中点击 `编辑`，修改资料后点击 `保存`。

可修改内容包括基础资料、显示名、编号、种子序号等。

### 4.3 停用选手

点击选手右侧 `停用`。

停用后，该选手不会再出现在当前项目的活跃选手列表中。已有历史比赛和积分记录不会被删除。

### 4.4 查看选手历史

点击选手右侧 `历史`，可查看：

- 最近比赛历史
- 对手
- 比分
- 胜负结果
- Elo 变化记录
- WCC 获分记录

## 5. CSV 导入导出

### 5.1 导入选手

在 `导入导出` 面板的文本框中粘贴 CSV。

推荐格式：

```csv
name,displayName,code,nickname,gender,birthDate,country,region,club,contact,avatarUrl,note,seedRank
张三,张三,P001,三哥,男,2000-01-01,中国,上海,一队,zhangsan@example.com,,左手,1
```

最常用的简化格式：

```csv
name,displayName,code,seedRank
张三,张三,P001,1
李四,李四,P002,2
```

点击 `导入选手`。

导入时系统会先自动创建 `pre-import-...sqlite` 备份，避免导入错误后无法恢复。

### 5.2 导出选手

点击 `选手 CSV` 下载当前项目选手资料。

### 5.3 导出排名

点击 `排名 CSV` 下载当前项目排名数据。

## 6. WCC 规则

WCC 分类似网球巡回赛积分：根据赛事等级和最终名次发放积分，并随时间过期或衰减。

### 6.1 默认规则

创建项目后会自动生成默认 `WCC 100` 规则。

默认名次积分包括：

- 冠军
- 亚军
- 四强
- 八强
- 十六强
- 参与分

### 6.2 创建或修改规则

在 `WCC 规则` 面板填写：

- `规则名称`
- `赛事等级`
- 冠军、亚军、四强、八强等分值
- 衰减方式

点击 `保存`。

已有规则可点击 `编辑` 后修改，或点击 `停用`。

### 6.3 衰减方式

支持三种方式：

- `固定过期`：在有效天数内保留全部分数，超过有效天数后归零。
- `线性削减`：在满额天数内保留全部分数，之后线性下降，到归零天数时变为 0。
- `阶梯削减`：按配置的天数和倍率分段削减。

## 7. 创建赛事

在 `赛事` 面板填写：

- `赛事名称`
- 赛制
- `赛事等级`，例如 `WCC_100`
- 开始时间
- 结束时间
- 报名截止时间
- 地点
- 主办方
- 说明
- 是否启用 Elo
- 是否启用 WCC

点击 `创建赛事`。

创建后，赛事会自动选中，并显示赛事设置、参赛名单和赛事操作按钮。

## 8. 参赛名单

### 8.1 同步项目选手

创建赛事后，点击 `同步选手`，系统会把当前项目内所有活跃选手加入赛事。

如果某些选手已经在赛事中，系统会跳过，不会重复添加。

### 8.2 修改参赛信息

在参赛名单中可修改：

- 种子
- 是否签到
- 报名状态：已报名、已签到、退赛

赛事抽签前可点击 `移除` 删除参赛选手。

抽签后，参赛名单会锁定，不能再随意移除或修改影响编排的数据。

## 9. 自动抽签与赛制

点击 `自动抽签` 后，系统会根据赛事赛制生成对阵。

### 9.1 淘汰赛

适用于单败淘汰。

使用方式：

1. 创建赛事时选择 `淘汰赛`。
2. 同步或添加参赛选手。
3. 点击 `自动抽签`。
4. 系统生成完整淘汰赛签表。
5. 录入每场成绩后，胜者自动进入下一轮。
6. 所有比赛完成后点击 `完成赛事`。

如果人数不是 2 的幂，系统会自动安排轮空。

### 9.2 循环赛

适用于所有选手互相比赛。

使用方式：

1. 创建赛事时选择 `循环赛`。
2. 同步参赛选手。
3. 点击 `自动抽签`。
4. 系统生成所有轮次。
5. 逐场录入成绩。
6. 查看 `成绩表` 和循环赛交叉表。
7. 全部完成后点击 `完成赛事`。

### 9.3 瑞士轮

适用于人数较多、不能所有人互相比赛的赛事。

使用方式：

1. 创建赛事时选择 `瑞士轮`。
2. 同步参赛选手。
3. 点击 `自动抽签` 生成第一轮。
4. 录入第一轮所有成绩。
5. 点击 `下一轮`，系统按当前成绩重新编排下一轮。
6. 重复录入成绩和生成下一轮。
7. 达到计划轮次或需要结束时，点击 `完成赛事`。

注意：生成下一轮前，应确保当前轮可比赛场次都已录入成绩。

### 9.4 杯赛

杯赛由小组赛加淘汰赛组成。

使用方式：

1. 创建赛事时选择 `杯赛`。
2. 设置 `小组数量` 和 `每组晋级`。
3. 同步参赛选手。
4. 点击 `自动抽签` 生成小组赛。
5. 录入小组赛成绩。
6. 点击 `淘汰阶段`，系统根据小组排名生成淘汰赛。
7. 录入淘汰赛成绩。
8. 全部完成后点击 `完成赛事`。

## 10. 对阵与成绩录入

在 `对阵与成绩` 面板中，每场比赛会显示：

- 轮次
- 选手 A
- 选手 B
- 台号
- 开赛时间
- 比赛状态
- 大比分
- 小局比分
- 结果类型

### 10.1 修改赛程信息

比赛未完成时，可修改：

- `台号`
- 开赛时间
- 状态：待赛、进行中、取消

### 10.2 录入成绩

默认大比分为：

```text
1 : 0
```

可选择结果：

- A 胜
- B 胜
- 平局
- A 弃权
- B 弃权
- 双弃权
- 取消

小局比分输入示例：

```text
11-8,9-11,11-6
```

点击 `录入` 后：

- 比赛变为已完成。
- Elo 会即时更新。
- 成绩表会刷新。
- 已完成比赛会显示 `撤销` 按钮。

### 10.3 撤销成绩

点击已完成比赛的 `撤销`。

撤销后：

- 该场比赛恢复为待赛。
- 小局比分会清空。
- 淘汰赛中依赖该场胜者的后续对阵会同步清理，需要重新录入。
- 可点击 `重算积分` 修正项目积分。

## 11. 完成赛事与积分

当赛事所需比赛都完成后，点击 `完成赛事`。

完成后：

- 赛事状态变为 `COMPLETED`。
- 系统根据成绩表确定最终名次。
- 如果赛事启用 WCC，会按赛事等级对应的 WCC 规则发放积分。
- WCC 排名会显示有效分和原始分。
- Elo 排名显示当前 Elo 和比赛场次。
- 综合排名结合 Elo 和 WCC 排名计算。

## 12. 成绩表与排名

### 12.1 成绩表

赛事内 `成绩表` 显示：

- 名次
- 选手
- 积分
- 胜平负
- Buchholz 等辅助分

### 12.2 综合排名

综合排名用于同时参考 Elo 和 WCC。

显示：

- 综合名次
- 选手
- 综合分
- Elo 名次
- WCC 名次

### 12.3 Elo 排名

显示：

- 当前 Elo
- 已赛场次

Elo 参考国际象棋规则，胜负双方根据赛前分差和项目 K 值调整。

### 12.4 WCC 排名

显示：

- 当前有效 WCC
- 原始 WCC

有效 WCC 会受到规则中的过期或衰减设置影响。

## 13. 导出赛事数据

选择赛事后，在 `对阵与成绩` 面板可下载：

- `对阵 CSV`：导出该赛事所有比赛、轮次、选手、比分和小局。
- `成绩 CSV`：导出该赛事成绩表。

这些链接需要管理员登录后使用。

## 14. 备份与恢复

### 14.1 自动每日备份

服务启动时，系统会检查当天是否已有自动备份。

如果没有，会创建：

```text
auto-daily-YYYY-MM-DDTHH-mm-ss.sqlite
```

同一天重复启动不会重复创建多份每日备份。

### 14.2 手动创建备份

在 `备份` 面板点击 `创建备份`。

成功后会出现：

- 备份文件名
- 文件大小
- 下载链接
- 恢复按钮

### 14.3 下载备份

点击备份记录中的 `下载`，可下载 SQLite 数据库文件。

### 14.4 恢复备份

点击备份记录中的 `恢复`。

恢复前系统会自动创建 `pre-restore-...sqlite` 备份。恢复完成后，建议重启服务。

## 15. 推荐比赛工作流

### 15.1 第一次使用

1. 启动软件。
2. 初始化管理员。
3. 创建项目。
4. 添加或导入选手。
5. 检查 WCC 规则。
6. 创建赛事。
7. 同步选手。
8. 自动抽签。
9. 录入成绩。
10. 完成赛事。
11. 查看排名。
12. 创建备份。

### 15.2 比赛现场

1. 提前创建项目和选手。
2. 比赛当天启动服务。
3. 创建当天赛事。
4. 同步选手并检查签到。
5. 抽签。
6. 逐轮录入成绩。
7. 让其他设备通过局域网访问页面查看赛程和成绩。
8. 完赛后创建备份并导出 CSV。

## 16. 常见问题

### 16.1 页面打不开

检查终端是否显示：

```text
Competition Manager listening on http://127.0.0.1:3001
```

如果端口被占用，可在 `.env` 中修改 `PORT`。

### 16.2 局域网设备访问不了

检查：

- `HOST=0.0.0.0`
- 防火墙是否放行
- 访问设备是否和电脑在同一局域网
- 使用的是电脑局域网 IP，不是 `127.0.0.1`

### 16.3 不能添加或修改数据

确认已经登录管理员账号。未登录时，写操作按钮会禁用。

### 16.4 自动抽签失败

检查：

- 是否已选择赛事。
- 是否已经同步或添加参赛选手。
- 赛事是否已经取消或完成。
- 抽签后参赛名单会锁定，不能随意移除选手。

### 16.5 无法完成赛事

通常是因为仍有必须完成的比赛没有录入成绩。

处理方式：

1. 检查 `对阵与成绩` 中是否还有 `录入` 按钮。
2. 录完所有必要比赛。
3. 再点击 `完成赛事`。

### 16.6 积分看起来不对

可点击左侧项目设置中的 `重算积分`。

重算会根据当前项目内所有已完成比赛和 WCC 事件重新计算，不会影响其他项目。

## 17. 用户视角测试记录

本轮文档编写前，已使用独立测试数据库从页面完成以下操作：

- 首次初始化管理员。
- 创建项目。
- 手动添加 4 名选手。
- 创建瑞士轮赛事。
- 同步参赛名单。
- 自动抽签生成第一轮。
- 录入第一轮成绩。
- 生成瑞士轮第二轮。
- 录入第二轮成绩。
- 完成赛事并发放 WCC。
- 查看成绩表、Elo 排名、WCC 排名、综合排名。
- 创建手动备份。
- CSV 导入选手。

测试中发现并修复的问题：

- 无请求体 POST 操作会被服务端拒绝，导致自动抽签、创建备份等页面操作失败。已修复为只有存在 JSON body 时才发送 JSON Content-Type。
- 新生成对阵的比分输入框显示 `null`。已修复为未录入比赛默认显示 `1` 和 `0`。
